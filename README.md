# TaskbarLyrics Light

原生 WPF 轻量版（当前 **v1.2.7**）。Light 版与原版共享 `TaskbarLyrics.Core` 歌词检索逻辑，但歌词窗、设置页、托盘菜单和诊断窗口都使用原生 WPF 实现，不依赖 WebView2 / Chromium 渲染栈，目标是更低进程数、更低内存占用，以及更贴近任务栏常驻场景的交互。

## 与原版对比

### 架构

| 项目 | 原版 (`TaskbarLyrics.App`) | 轻量版 (`TaskbarLyrics.Light`) |
|------|---------------------------|-------------------------------|
| 歌词渲染 | WebView2 + HTML/CSS/JS | 原生 WPF `LyricsDisplayControl` |
| 设置页 | WebView2 加载 `settings.html` | 原生 WPF `SettingsWindow` + `SettingsTheme.xaml` |
| UI 框架 | WPF + WPF-UI | 纯 WPF |
| WebView2 Runtime | 必须 | 不需要 |
| 默认字体 | 内置 Source Han Sans SC | 内置 Source Han Sans SC |
| 歌词业务逻辑 | `TaskbarLyrics.Core` | 共享 `TaskbarLyrics.Core` |
| 歌词缓存 / 歌曲映射 | `%APPDATA%\TaskbarLyrics\` | 与原版共享 |
| 设置与单实例 | 原版独立 | Light 版独立 |

### 性能（历史实测）

测试环境：Windows 11 x64（22631），`Release` 发布，`win-x64` 框架依赖（`--self-contained false`），启动后静置 **20 s** 取样；内存为**主进程及其子进程树**合计（原版含 WebView2）。测试日期：**2026-07-01**；版本：原版 **v1.1.1** / 轻量版 **v1.1.0**。

| 指标 | 原版 v1.1.1 | 轻量版 v1.1.0 | 变化 |
|------|-------------|---------------|------|
| 发布包体积 | 75.7 MB（54 文件） | **67.6 MB**（38 文件） | **约 -11%** |
| 进程数（含子进程） | **7**（1 主进程 + 6x WebView2） | **1** | **单进程** |
| 工作集内存 | **~669 MB** | **~340 MB** | **约 -49%** |
| 专用内存 Private | **~398 MB** | **~234 MB** | **约 -41%** |

> Light 版仍内置思源黑体，所以磁盘体积不会极限缩小；真正减少的是 Chromium 多进程栈、跨进程脚本调用和 WebView2 Runtime 依赖。

## 功能亮点

### 歌词与播放源

- 支持 QQ 音乐、网易云音乐、酷狗音乐、Spotify 的 SMTC 识别
- 播放源可启用/关闭，并支持拖拽调整识别优先级
- 本地歌词支持 `.lrc` / `.qrc` / `.krc`
- 本地歌词策略：
  - 本地优先
  - 在线失败后使用
- 全局歌词偏移，以及 QQ 音乐 / 网易云 / 酷狗 / Spotify 独立偏移
- 支持歌词翻译显示开关；歌词源提供翻译时，译文会跟随原文显示
- 翻译可设置同行显示 / 译文换行、译文字号比例和译文透明度
- 数值设置支持直接输入，也支持 `+` / `-` 按钮步进
- SMTC 时间轴稳定器，减少部分播放器时间轴跳变导致的歌词滚动进度错误
- 歌词命中诊断、重新匹配、缓存清理

### 封面

- SMTC 专辑封面异步读取和交叉淡入
- 本地封面回退：
  - 同目录图片
  - 音频内嵌封面
- 封面解码兜底：
  - WPF `BitmapImage`
  - Windows `BitmapDecoder`
  - ImageSharp
- 本地/在线封面策略：
  - 在线优先
  - 本地优先
  - 仅在线
  - 仅本地
- 歌词封面显示开关，默认开启
- 封面柔光开关和强度调节，默认关闭
- 封面样式：
  - 方形
  - 圆角方形
  - 圆形
  - 隐藏封面
- 封面大小调节，横向排列和上下两行布局都生效
- 封面布局：
  - 横向排列：封面在左，歌词/频谱在右
  - 上下两行：封面在上，歌词/频谱在下，并保持下方内容的底部位置不变
- 上下两行布局可单独调整：
  - 歌名/歌手信息显示开关
  - 封面到歌名/歌手信息间距
  - 封面到内容间距
  - 上方封面 X 偏移
  - 上方封面 Y 偏移
  - 歌词/频谱 X 偏移
  - 歌词/频谱 Y 偏移
- 上下两行布局会把封面、歌名/歌手信息、歌词/频谱和 X 偏移一起纳入自动宽度计算
- 可用封面主色给歌词和频谱取色

### 频谱

- 24 条实时频谱
- 频谱总开关
- 纯音乐时显示频谱
- 未检索到歌词时显示频谱（默认开启）
- 有歌词时也显示频谱（默认关闭）
- 频谱样式：
  - 中心扩散
  - 底部柱状
  - 镜像波形
  - 细线
  - 点阵
  - 呼吸条
- 频谱配色：跟随文字、跟随封面、轻量渐变
- 频谱调参窗口，可调整响应速度、高度范围、透明度和刷新间隔
- WASAPI 环回采集按需启动，仅在需要显示频谱时采集音频

### 窗口与动画

- 双行歌词显示
- 歌词切换动画：
  - 上滑
  - 无动画
- 旧配置中的淡入淡出和紧凑滑动会自动归一到上滑
- 动画强度：减少动画、标准、丝滑
- 使用 `CompositionTarget.Rendering` 驱动歌词过渡、颜色、进度和频谱插值，动画更贴近屏幕刷新
- 影响窗口布局的设置会即时落点，避免宽高、封面布局、译文排版等变化产生拖拽式突变
- 上下两行布局会为歌词切换动画保留垂直裁剪缓冲，减少第二行上滑时文字只显示半截的问题
- 自动窗口宽度，随歌词内容变化；上下布局会额外计算封面、歌名/歌手信息和 X 偏移
- 自动窗口高度，随字号、封面布局和歌词区域变化；上下两行布局增加高度时优先向上展开，底部锚点不变
- 可手动设置窗口宽度、高度、自动尺寸偏移
- 水平锚点：左侧 / 居中 / 右侧
- 目标屏幕：主屏幕、鼠标所在屏幕、指定屏幕序号
- X / Y 偏移
- 强制置顶开关
- 从 Alt+Tab 隐藏（`WS_EX_TOOLWINDOW`）
- 背景显示开关、背景不透明度、背景材质、边框、文字效果（无、阴影、描边、柔光）
- 歌词颜色可检测背景自动变色：开启歌词背景时按背景材质判断，透明时按窗口所在屏幕区域判断
- 歌曲进度显示（默认关闭）：
  - 底部细线
  - 封面进度环
  - 封面底边
  - 时间胶囊
  - 呼吸进度点
  - 边框进度环
  - 背景进度
- 旧配置中的歌词下划线和频谱底线会自动归一到底部细线
- 歌曲进度可调颜色（跟随歌词、跟随封面、自定义）、粗细和透明度，也可固定进度条长度并设置左/中/右锚点
- 封面进度样式在隐藏封面时自动回退到底部细线

### 设置增强

- 设置页支持隐藏不可调项目，默认隐藏；也可切换为置灰显示
- 右侧设置区顶部提供真实歌词控件实时预览，滚动设置项时预览保持可见
- 可按播放器单独配置进度样式和频谱样式，默认关闭
- 关于页支持导入 / 导出 Light 设置 JSON

### 托盘与启动

- 单实例，二次启动会激活已有实例并打开设置
- 系统托盘菜单，支持悬浮二级菜单直接设置频谱、进度、封面、文字效果和切换动画；二级菜单可关闭对应功能，选择具体样式时会自动开启
- 如果出现歌词不匹配、翻译未出现或其他歌词相关异常，可右键托盘图标点击 `重新匹配` 重新检索并刷新当前歌曲的歌词结果
- 开机自启动（当前用户注册表 `Run` 项，独立名称 `TaskbarLyrics.Light`）
- 播放器联动：
  - 检测到已启用播放器开始播放时自动显示歌词
  - 全部播放器停止或关闭后自动隐藏歌词
  - 启动阶段若开启“播放器关闭时隐藏”，默认不闪出歌词窗

### 更新检查

- 关于页支持检查 GitHub Release，并可下载 Light 更新包后自动覆盖安装
- 优先使用 GitHub API，并从 Release assets 中识别 Light 更新包
- Light 更新包文件名建议使用 `TaskbarLyrics_light_v1.2.7.zip`；不带 `.zip` 后缀但内容为 zip 的 `TaskbarLyrics_light_v1.2.7` 也会尝试识别
- 点击“下载并安装更新”后，会下载到本地临时目录、解压、退出当前进程、覆盖安装目录并自动重启；安装目录无写入权限时会请求管理员权限
- API 失败时回退到 `releases/latest` 跳转地址解析，降低部分网络环境下误报“网络不行”的概率
- 自动检查更新每天最多一次

## 性能优化

- 延迟初始化 SQLite / EF Core 与歌词同步服务
- 在线歌词 Provider 懒加载
- 统一 16 ms 前端帧定时器，歌词和频谱按不同节奏调度
- 频谱按需启动 WASAPI 环回采集
- 行高与文本宽度测量缓存
- WPF 画刷和封面 Bitmap 冻结，减少跨线程和渲染开销
- 设置页字体列表延迟枚举
- 封面读取失败会记忆拒绝签名，避免同一坏图反复解码

## 系统要求

- Windows 10/11 x64
- 从源码运行或框架依赖发布时，需要安装 [.NET 8 Runtime](https://dotnet.microsoft.com/download/dotnet/8.0)
- 不需要 Microsoft Edge WebView2 Runtime

## 运行

```bash
dotnet run --project TaskbarLyrics.Light/TaskbarLyrics.Light.App
```

## 发布

推荐使用 Light 打包脚本，默认独立发布并生成 `publish/TaskbarLyrics_light_v1.2.7.zip` 这种格式的更新包：

```powershell
.\TaskbarLyrics.Light\scripts\publish-light.ps1
```

如需框架依赖发布（目标机器需安装 .NET 8），使用：

```powershell
.\TaskbarLyrics.Light\scripts\publish-light.ps1 -FrameworkDependent
```

脚本会读取 `TaskbarLyrics.Light.App.csproj` 中的 `<Version>`，发布目录名和 zip 文件名统一为 `TaskbarLyrics_light_v{版本}`。发布包请完整解压后运行 `TaskbarLyrics.Light.exe`，需要保留 `Assets` 等目录。

## 配置与数据目录

| 用途 | 路径 |
|------|------|
| Light 设置 | `%APPDATA%\TaskbarLyrics.Light\settings.json` |
| 单实例锁 | `%LOCALAPPDATA%\TaskbarLyrics.Light\TaskbarLyrics.Light.lock` |
| 运行日志 | 程序目录下 `Logs\app_debug.log` |
| 歌词缓存 | `%APPDATA%\TaskbarLyrics\cache`（与原版共享） |
| 歌曲映射数据库 | `%APPDATA%\TaskbarLyrics\database\song_maps.db`（与原版共享） |

两版可以同时安装；设置文件和单实例锁独立，歌词缓存与歌曲映射数据库共享。

## 选用建议

- 优先 Light 版：日常任务栏常驻、在意内存和进程数、不想依赖 WebView2、需要开机自启、播放器联动、原生设置页或自动更新安装、更多与原版不同的功能
- 优先原版：希望完全跟随主仓库 WebView2 设置页体验
