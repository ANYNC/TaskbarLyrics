# TaskbarLyrics 设置页交互原型改版设计

## 目标与范围

本次只制作独立 HTML 交互原型，用于评审新版设置页的信息架构、视觉和交互；不修改 `TaskbarLyrics.App` 正式设置页，不连接 WebView2 消息接口，也不持久化设置。

目标是在 1180×760 的默认窗口中消除超长单页与导航状态冲突，让高频设置更快定位，并通过实时任务栏歌词预览降低外观参数的理解成本。

## 设计方向

采用“桌面工具型分页面”结构：左侧导航切换独立内容页，右侧只呈现当前分类。外观和窗口页使用“设置区 + 固定预览区”，其余页面使用单列紧凑设置组。

视觉保持深色专业工具风格，使用中性黑灰底色与统一蓝色强调色。减少多层大卡片嵌套，以分组标题、细分隔线和适度底色建立层级。

## 信息架构

导航宽度为 220px，可折叠为图标栏。分类为：

1. 播放源：播放器启用、识别优先级、本地歌词。
2. 歌词：启动显示、翻译、频谱及其显示模式。
3. 外观：字号、字体、字重、颜色、阴影、封面尺寸与圆角。
4. 窗口：背景、边框、不透明度、宽度、锚点与置顶。
5. 常规：开机启动、自动更新等普通应用行为。
6. 高级：SMTC 监视器、频谱调试、缓存清理。
7. 关于：版本、许可证、项目链接和检查更新。

顶部标题和说明随导航切换。右上角显示即时保存状态与低权重“恢复默认”按钮。

## 播放源扩展性

播放源不得写死为四个固定列。原型使用统一播放源数据模型生成卡片，每项包含稳定 ID、显示名称、适配器名称、图标、启用状态、可用状态和识别优先级。

卡片区采用响应式网格：默认每张卡片最小宽度 180px，通过 `repeat(auto-fit, minmax(...))` 自动换行。新增播放源只增加数据项，不修改布局结构。大量播放源时列表区域独立滚动，页面标题、本地歌词设置和主要操作保持可见。

每张卡片提供明确的复选控件和启用状态；缺少专用图标时显示统一首字母占位图标；不可用的适配器显示原因并禁用操作。启用的来源进入“识别优先级”列表，支持上移和下移，避免依赖拖拽才能操作，并兼容键盘使用。

## 页面与组件

### 播放源

- 可扩展播放器卡片网格。
- 启用来源的识别优先级列表。
- 本地歌词开关、目录输入区和“浏览”按钮。
- 关闭本地歌词后，目录区域禁用但不清空内容。

### 歌词

- 使用真实 Switch 控件表达布尔状态。
- 关闭频谱时，频谱显示模式禁用并降低透明度。
- 子设置缩进并使用细蓝色引导线表达依赖关系。

### 外观与窗口

- 左侧为紧凑设置组，右侧为固定预览区。
- 预览包含桌面背景、任务栏和歌词浮窗，不模拟完整操作系统。
- 字号、字体、字重、颜色、阴影、封面大小、间距、圆角、背景、边框、不透明度、宽度和锚点即时更新预览。
- “安全范围”改写为“限制在任务栏适配范围”，并提供明确说明。

### 常规、高级与关于

- 开机启动和自动更新放入常规页。
- 调试工具和缓存清理放入高级页。
- 清除缓存使用危险次级按钮，并要求二次确认。
- 关于页不再重复出现在其他页面底部。

## 视觉规范

- 默认窗口：1180×760，可响应缩放。
- 侧栏：220px；折叠后 68px。
- 控件高度：36px；常规设置行约 58px。
- 原型默认使用系统字体 `Microsoft YaHei UI, Microsoft YaHei, sans-serif`，避免依赖只有单字重的内置思源黑体。
- 字重只使用 400、600、700 三档；若运行环境无法提供对应字重，则由系统字体回退处理，不合成额外中间字重。
- 使用一致的 16px 线性 SVG 图标，避免混用 Unicode 字符。
- 主强调色用于选中、焦点和有效状态；危险色只用于破坏性操作。
- 所有交互控件提供 hover、active、focus-visible、disabled 状态。

## 交互与状态

原型使用页面内 JavaScript 状态对象。导航切换、播放器启用、优先级调整、开关、下拉框、数值步进和范围滑块均更新状态；外观和窗口相关状态同时更新预览。

修改后顶部显示短暂“已保存”反馈，但只表示原型内状态已更新。刷新页面恢复默认演示数据。

“恢复默认”打开确认对话框，确认后重置所有演示状态；取消不产生修改。依赖设置在父开关关闭时仅禁用，不丢弃原值。

## 响应式与可访问性

- 低于 980px 时，外观和窗口页的预览移动到设置区下方。
- 播放源卡片根据可用宽度自动调整列数。
- 使用真实 `button`、`input`、`select` 和 `dialog`；Switch 提供可访问名称和状态。
- 主要操作支持 Tab 导航，焦点样式清晰；优先级调整不依赖拖拽。
- 动画尊重 `prefers-reduced-motion`。

## 验证标准

- 默认尺寸下每次只显示一个分类，不再纵向堆叠全部页面。
- 播放源数量从 1 增加到 12 时无需修改网格 CSS 或卡片结构。
- 关闭频谱、背景或本地歌词后，对应子设置正确禁用。
- 外观和窗口控件能即时反映到预览。
- 恢复默认和清除缓存均有确认流程。
- 主要控件可通过键盘访问，缩窄窗口后不存在水平溢出。

## 非目标

- 不接入真实播放器扫描、文件夹选择器或系统设置。
- 不实现配置持久化、更新检查、缓存删除和调试窗口。
- 不修改正式应用中的 HTML、CSS、JavaScript、XAML 或 C#。
