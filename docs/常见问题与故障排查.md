# TaskbarLyrics 常见问题与故障排查

## 1. 歌词不滚动

根据播放器逐项排查：

- 如果是网易云音乐播放器，需要安装[inflink-rs](https://github.com/apoint123/inflink-rs)插件获取完整SMTC支持。
- 如果是酷狗音乐版本，其提供的播放进度不完整只能识别歌曲而无法稳定滚动歌词，可以尝试兼容性较好的第三方播放器 [MoeKoeMusic](https://github.com/MoeKoeMusic/MoeKoeMusic)。
- 如果是未适配播放器，可以在任务栏歌词工具托盘菜单中打开SMTC监视器，查看`Extrapolated`时间是否正常推进，如果正常推进仍不滚动歌词可题issue反馈。

## 2. 显示了错误歌曲的歌词

由于歌手别名等原因，可能导致识别不准确，可提issue并将安装目录Log中最近的日志上传，以便作者持续优化检索策略。

## 3. 为什么没有歌词翻译

翻译歌词取决于具体歌词来源。即使原歌词存在，也不一定能获得翻译。

请确认“设置 → 歌词 → 显示歌词翻译”已经开启。如果仍然没有翻译，通常表示当前歌词来源没有提供可用翻译，不代表程序发生故障。

## 4. 本地歌词没有被识别

请检查：

1. 已开启“设置 → 播放源 → 优先使用本地歌词”。
2. 已添加包含音乐或歌词的本地目录。
3. 歌词文件格式为 `.lrc`、`.qrc` 或 `.krc`。
4. 歌词文件与音频文件使用相同或接近的文件名。
5. LRC 文件包含时间标签，而不是只有普通文本。
6. 目录和文件具有正常的读取权限。

添加或修改本地目录后，可以重新播放歌曲，让程序重新匹配。

### 音频文件中的内嵌歌词

程序会扫描本地目录中的 `.mp3`、`.flac`、`.m4a`、`.aac`、`.wav`、`.ogg`、`.opus` 和 `.wma` 音频文件，并尝试读取其中的内嵌歌词。当前兼容范围有以下限制：

- 只尝试读取名为 `LYRICS`、`SYNCEDLYRICS` 或 `UNSYNCEDLYRICS` 的歌词标签。
- 标签内容需要采用带时间标记的 LRC 形式，例如 `[01:23.45]歌词内容`，才能形成可滚动的歌词。
- 不同音乐管理软件可能使用其他 ID3 或 MP4 标签写入歌词，这些方式目前不一定能被识别。
- 音频文件名仍应尽量包含正确的歌曲名和歌手名，便于与播放器当前歌曲匹配。

如果内嵌歌词无法识别，最稳妥的做法是把歌词导出为带时间标签、与音频同名的 `.lrc` 文件，放在已添加的本地目录中。

## 5. 歌曲封面不显示

封面通常来自播放器提供的歌曲信息。对于已配置的本地音乐目录，程序也会尝试读取：

- 音频文件中的内嵌封面
- 与音频同名的图片
- `cover`、`folder`、`front`、`album` 等常见封面文件

没有找到封面时，歌词窗口会显示播放器对应的字母或音乐符号，这属于正常回退效果。

## 6. 关闭设置窗口后，为什么还能看到 WebView2 进程

TaskbarLyrics 主版本使用 WebView2 显示设置和歌词界面。关闭设置窗口后，设置页资源会被释放；但只要 TaskbarLyrics 和歌词窗口仍在运行，任务管理器中保留若干 `msedgewebview2.exe` 进程属于正常现象。

如果希望全部结束，请从托盘菜单退出 TaskbarLyrics。若退出后进程仍长期存在，再附任务管理器截图和日志反馈。

## 7. 如何彻底卸载

1. 从托盘菜单退出 TaskbarLyrics。
2. 删除解压后的程序目录。
3. 如果不需要保留设置、缓存和单曲偏移，再删除 `%APPDATA%\TaskbarLyrics`。
4. 如果启用了开机启动，建议在删除程序目录前先关闭该选项。

## 8. 如何提交有效的问题反馈

提交 Issue 时，建议提供：

- Windows 版本
- TaskbarLyrics 版本
- 播放器名称和版本
- 歌曲标题与歌手
- 问题发生时的表现
- 是否每次都能复现
- 程序目录 `Logs` 文件夹中的相关日志

请在发送日志前检查其中是否包含你不希望公开的歌曲名称或本地文件路径。
